# 🐾 OpenClaw 自动恢复机制

## 自动检测，不再怕漏！

现在从仓库拉取备份后，系统会**自动检测**并**自动修复**缺失的插件。

---

## 🔄 工作原理

### 方案 1：Git Hook 自动触发（推荐）

当你运行 `git pull` 或 `git checkout` 时，git 会自动触发 `post-checkout` hook：

```bash
git pull origin main
# ↓ 自动触发
# .git/hooks/post-checkout
# ↓ 自动运行
# scripts/auto-check-restore.sh
# ↓ 如有需要，自动运行
# scripts/restore-config.sh
```

**完全自动！** 你只需要 pull，剩下的交给系统。

---

### 方案 2：手动运行

如果 git hook 没有触发，可以手动运行：

```bash
cd ~/.openclaw/workspace
./scripts/auto-check-restore.sh
```

或者直接运行恢复脚本：

```bash
./scripts/restore-config.sh
```

---

## 📋 脚本说明

| 脚本 | 用途 | 运行方式 |
|------|------|----------|
| `auto-check-restore.sh` | 自动检查 + 自动修复 | git hook 自动调用 / 手动运行 |
| `restore-config.sh` | 完整恢复流程 | 手动运行 / 被 auto-check 调用 |
| `backup-plugins-manifest.sh` | 备份时生成清单 | 备份前手动运行 |

## 🎯 使用场景

## ⚙️ Git Hook 说明

**位置：** `.git/hooks/post-checkout`

**触发条件：**
- `git pull` 后
- `git checkout` 后
- `git clone` 后（首次 checkout）

**运行方式：** 静默模式（只在需要修复时输出）

**注意：** 如果删除了 `.git` 目录重新 clone，需要重新设置 hook：

```bash
# 重新 clone 后
cp scripts/post-checkout-template .git/hooks/post-checkout
chmod +x .git/hooks/post-checkout
```

或者手动运行一次：

```bash
./scripts/auto-check-restore.sh
```

---

## 🔍 检测逻辑

自动检查脚本会：

1. **扫描配置文件** - 提取需要的频道插件
2. **检查已加载插件** - 对比配置和实际状态
3. **发现缺失** - 自动运行恢复脚本
4. **安装插件** - 从 npm 安装并复制到 extensions
5. **重启 Gateway** - 使配置生效

## 🛡️ 安全保障

- ✅ 只在检测到缺失时才运行恢复
- ✅ 不会覆盖已有配置
- ✅ 静默模式适合 git hook 调用
- ✅ 失败不会中断 git 操作
