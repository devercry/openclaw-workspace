## 🤖 OpenClaw-RL Tinker 云端训练配置

**时间：** 16:19 - 17:12  
**项目：** https://github.com/Gen-Verse/OpenClaw-RL

### 背景
用户想在本地训练 AI 助手，但机器没有 NVIDIA GPU，改用 Tinker 云端方案。

### 配置进度
| 步骤 | 状态 | 说明 |
|------|------|------|
| 克隆项目 | ✅ 完成 | `/home/tenbox/OpenClaw-RL` |
| Tinker API Key | ✅ 已提供 | 用户于 16:19 提供 |
| 依赖安装 | ❌ **放弃** | Python 3.13 兼容性问题太多，依赖太大 |
| 官方扩展 | ✅ **完成** | rl-training-headers 已安装并启用 |

### 遇到的问题
**Python 版本兼容性：**
- 当前 Python：3.13.5
- `nvidia-modelopt` 和 `nvidia-resiliency-ext` 仅支持 Python ≤3.12
- `numpy==1.26.4` 与 Python 3.13 不兼容（需要 >=2.1.0）
- 依赖包太大（PyTorch 900MB, sgl_kernel 535MB），安装非常慢

### 解决方案
**改用官方扩展 rl-training-headers：**
- 复制到 `/home/tenbox/.openclaw/extensions/rl-training-headers`
- 启用插件：`openclaw plugins enable rl-training-headers`
- 重启 Gateway

### 扩展功能
该扩展会在每个 LLM API 请求中注入两个 HTTP 头：
- `X-Session-Id`: 会话 ID
- `X-Turn-Type`: `main`（用户对话）或 `side`（心跳/记忆等）

这样后端的 RL 训练系统可以：
- 按会话 ID 分组，重建完整对话
- 过滤 `main` 类型的对话用于奖励建模
