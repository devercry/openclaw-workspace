# Memory Log - 2026-03-25

## Website Design Project - CSS Variable Compatibility Issue

**Time:** 14:00 - 14:23

### Problem Discovered
Frontend Developer subagent updated `variables.css` with new variable naming scheme, but `base.css` and `components.css` still used old variable names. This caused the website to display as a white screen.

### Variable Naming Mismatch
**Old variables (in base.css/components.css):**
- `--font-size-body-large`, `--font-size-body-medium`, `--font-size-body-small`
- `--font-size-title-medium`
- `--color-on-primary`, `--color-on-secondary`, `--color-on-surface`
- `--color-surface`, `--color-border-light`
- `--bg-primary`, `--text-primary` (in responsive.css)

**New variables (in updated variables.css):**
- `--font-size-lg`, `--font-size-base`, `--font-size-sm`
- `--font-size-2xl`
- `--color-text-primary`, `--color-bg-card`
- `--color-border`
- `--color-bg-primary`, `--color-text-primary`

### Solution Applied
Added compatibility layer to `variables.css` to map old variable names to new ones:

```css
/* Font size compatibility */
--font-size-body-small: var(--font-size-sm);
--font-size-body-medium: var(--font-size-base);
--font-size-body-large: var(--font-size-lg);
--font-size-title-medium: var(--font-size-2xl);

/* Color compatibility */
--color-on-primary: var(--color-text-primary);
--color-on-secondary: var(--color-text-primary);
--color-on-surface: var(--color-text-primary);
--bg-primary: var(--color-bg-primary);
--bg-secondary: var(--color-bg-secondary);
--text-primary: var(--color-text-primary);
```

Also updated `components.css` to use new variable names directly for:
- `.btn-primary`, `.btn-secondary`, `.btn-outline`, `.btn-text`
- `.card`, `.card-header`, `.card-footer`
- `.product-card`

### Lesson Learned
When multiple CSS files reference shared variables, ensure all files are updated consistently or maintain a compatibility layer to prevent breaking changes.
